# Module: Internationalization (i18n)

## Overview
App supports **Hindi (हिन्दी)** and **English** with runtime language switching. Default language based on device locale.

## Tech Stack
- **Library**: `i18next` + `react-i18next` + `expo-localization`
- **Storage**: Language preference saved in AsyncStorage

## Supported Languages

| Code | Language | Direction |
|------|----------|-----------|
| `en` | English | LTR |
| `hi` | हिन्दी (Hindi) | LTR |

## Language Detection Flow
```
1. App launch → check AsyncStorage for saved preference
2. If saved → use saved language
3. If not → check device locale via expo-localization
4. If device is hi/hi-IN → use Hindi
5. Else → default to English
```

## AI Agent Language
- AI agents respond in the user's selected language
- System prompt includes: `"Respond in {{language}}"`
- Health agent advice given in selected language

## Implementation Notes
- Use `i18next` with `react-i18next` for React Native
- Detect device locale with `expo-localization`
- Save preference to AsyncStorage key: `@cattlecare_language`
- All static text uses translation keys — no hardcoded strings
- Numbers, dates use locale formatting (`Intl.NumberFormat`, `Intl.DateTimeFormat`)
- Emoji and icons are language-neutral (no translation needed)
